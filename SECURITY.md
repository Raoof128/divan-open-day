# Security policy

## Supported versions

DIVAN has no production release or deployed public service. Maintainers still
accept private reports about the current repository because a defect may affect
future builds or deployment controls.

## Report a vulnerability

Use GitHub private vulnerability reporting from this repository's **Security**
tab. Select **Report a vulnerability** and include the affected commit, impact,
reproduction steps, and a minimal suggested fix when available.

Do not open a public issue, pull request, discussion, or commit containing an
unfixed vulnerability. Do not include credentials, private content, visitor
data, infrastructure identity, or unrelated personal information. Redact test
evidence to the minimum needed for maintainers to reproduce the problem.

The repository owner must enable private vulnerability reporting before making
the repository public. If the GitHub reporting form is unavailable, do not
publish the report in another repository channel.

## Scope

Reports may cover source code, build and content validation, browser privacy,
offline release coherence, static delivery headers, container isolation, or
deployment and rollback scripts. Production content approval, literary review,
and general event enquiries do not belong in the security reporting form.
